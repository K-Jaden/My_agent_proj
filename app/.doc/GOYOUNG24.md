# 고용24 (워크넷) 채용 공고 API 에이전트 활용 명세서

> **문서 목적:** 안티그래비티(Anti-Gravity) 등의 위키에 등록하여, AI 에이전트가 워크넷 API를 어떻게 이해하고 활용해야 하는지 개발자와 기획자가 공유하기 위한 문서입니다. 에이전트의 프롬프트나 System Instruction에 반영하기 좋은 구조로 작성되었습니다.

---

## 1. 도구 개요 (Tool Overview)
- **목적:** 구직자의 맞춤형 자소서 작성을 위해 특정 채용 공고의 상세 정보(기업 정보, 직무 내용, 우대 조건 등)를 실시간으로 조회합니다.
- **사용 시점:** 사용자가 특정 공고(구인인증번호)에 대한 자소서 작성을 요청하거나, 추천받은 공고를 선택했을 때 공고의 세부 스펙을 파악하기 위해 최우선으로 호출해야 합니다.

---

## 2. 요청 파라미터 (Request Parameters)
에이전트는 사용자의 대화나 앞선 검색 결과에서 **구인인증번호**만을 추출하여 전달합니다.

| 파라미터명 | 타입 | 필수여부 | 설명 | 에이전트 인지 사항 |
| :--- | :--- | :--- | :--- | :--- |
| `wantedAuthNo` | String | Y | **구인인증번호** <br>(예: K120612108130052) | 사용자의 입력값에서 추출하여 전달 |

> ⚠️ **개발 연동 참고 (에이전트 추상화):** > 실제 고용24 API 호출에 필요한 `authKey`, `callTp(D)`, `returnType(XML)`, `infoSvc(VALIDATION)` 등의 고정 파라미터는 백엔드 시스템(LangChain Tool 내부)에서 자동으로 매핑되도록 처리합니다. 에이전트에게는 오직 `wantedAuthNo`만 요구하여 환각(Hallucination)을 방지합니다.

---

## 3. 출력 결과 분석 가이드 (Output Data Extraction)
수많은 출력 항목 중, 에이전트는 자소서 작성에 필요한 **핵심 정보(★)**를 우선적으로 필터링하여 컨텍스트로 활용해야 합니다.

### 3.1 기업 기본 정보 (`<corpInfo>`)
- `<corpNm>`: 회사명
- `<indTpCdNm>`: 업종 (기업의 산업군 파악)
- **★ `<busiCont>`: 주요사업내용** (기업 분석 및 지원 동기 작성 시 활용)
- **★ `<homePg>`: 회사 홈페이지 주소** (추가 크롤링 도구 연계 시 타겟 URL로 활용)

### 3.2 채용 상세 및 직무 요구사항 (`<wantedInfo>`)
- `<wantedTitle>`: 구인제목
- **★ `<jobCont>`: 직무내용** (지원자가 입사 후 수행할 실제 업무. 역량 매칭의 최우선 기준점)
- `<enterTpNm>` / `<empTpNm>`: 경력조건 (신입/경력) 및 고용형태
- `<eduNm>` / `<major>` / `<certificate>`: 요구 학력, 전공, 자격면허
- **★ `<pfCond>` / `<etcPfCond>`: 우대조건 및 기타우대조건** (지원자의 차별화된 강점을 어필할 수 있는 핵심 포인트)
- `<keywordList> -> <srchKeywordNm>`: 공고 키워드 (해당 직무의 트렌드 키워드로 판단하여 자소서 문맥에 자연스럽게 반영)

---

## 4. 에이전트 핵심 행동 지침 (Agent Action Guidelines)

시스템 프롬프트(System Prompt)에 주입해야 할 에이전트의 구체적인 행동 규칙입니다.

1. **의미론적 매칭 (Semantic Matching):** 반환된 데이터 중 **`직무내용(<jobCont>)`**과 **`우대조건(<pfCond>)`**을 가장 중요하게 다루십시오. 사용자의 이력서(Resume) 내 경험과 해당 요구사항 간의 교집합을 찾아 논리적인 연결고리를 만드십시오.
2. **연속 도구 호출 (Tool Chaining):** API 반환값에 **`회사 홈페이지(<homePg>)`** URL이 존재할 경우, 심층적인 인재상 및 최근 기업 이슈 파악을 위해 즉시 `웹 크롤러 도구(Web Crawler Tool)`를 이어서 호출하십시오.
3. **근거 기반 생성 (Grounded Generation):** 자소서 내용은 반드시 조회된 API 데이터의 팩트(사업내용, 직무내용 등)에 기반하여 작성해야 합니다. 지원 공고와 무관한 직무나 기업 정보를 임의로 지어내지 마십시오.v
